# 音楽反応アバター 開発引き継ぎメモ

> このファイルは claude.ai 上での企画・設計の議論をまとめた引き継ぎ用ドキュメントです。
> Claude Code 側で最初に読み込ませて、文脈を共有する目的で使ってください。
> （例：リポジトリ直下に `PLAN.md` や `CLAUDE.md` として置く）

## このプロジェクトの目的

`rotejin/tomari-guruguru`（ブラウザアバター）の「マウスカーソル追従」と「表情」の仕組みを下敷きにして、

- 自作キャラへ差し替える
- 「トーク（口パク）」機能は使わない
- その代わりに「**音楽に反応して踊る／演出する**」機能を作る
- テーマは音楽・ダンス系
- スマホ（iPhone / Safari・Chrome）ブラウザでの動作を想定

## ベースリポジトリの構造（把握済み）

- 描画は Live2D / Spine のような骨格アニメではなく **画像差し替え式**。
- マウス追従＝**25方向**（5列×5行, `r{0-4}c{0-4}`）の顔差分画像を切り替えている。
- 表情/口パク＝**A〜F の6状態**（目開け/閉じ × 口とじ/中間/開け）の画像切り替え。
  - 画像パス例：`slices2/A/r2c2.webp`（正面・水平・目開け口とじ）
- 音声解析は **Web Audio API の `AnalyserNode` でマイク or 音声ファイルの音量を測る**配線（旧「トーク」版由来）。統合アプリ `avatar-app.jsx` に残してあり、現状は音量メーター表示のみ（口パク出力は無効化済み）。
  → この音声解析の配線が、そのまま「音楽反応」の土台になる。
- React + Babel + Google Fonts を CDN 読み込み。ローカルは `python -m http.server 8000` で起動。

## 技術的制約（iOS / Safari, 確認済み）

- **アプリ自身が曲を再生する → 最も安定。iOSでも完全に動作する。** ← これを主軸にする。
- マイクで周囲の音を拾うモード → 動くが「前面表示時のみ・許可が必要・音楽か雑音かの区別は別途必要」。サブ機能扱い。
- 他アプリ（Apple Music / Spotify 等）の再生を検知 → **不可**（Webからシステム音声や他アプリ状態は取得できない）。
- iOS では Chrome も含め全ブラウザが WebKit で動くため、Safari と挙動はほぼ同じ。

## 採用方針

1. **入力**：ユーザーが曲ファイルを読み込む（ドラッグ&ドロップ or ファイル選択）。同梱曲を用意してもよい。
2. **解析**：`AnalyserNode` の `getByteFrequencyData()`（帯域別）＋ `getByteTimeDomainData()`（音量/波形）。
3. **ビート検出**：低域（ベース/キック）のエネルギーを移動平均でならし、平均を一定割合超えたら「ビート」とみなす簡易方式。軽くて十分。
4. **反応の表現**：
   - 全体モーションを **CSS transform** で表現（`translateY` でバウンド / `scale` で拍動 / `rotate` で揺れ）＝**追加作画不要**。
   - 既存の25方向フレームをビートに合わせて切り替え、首振りダンスに見せる。
   - ビートごとに **音符パーティクル** を飛ばす。
   - 低域 → 大きく跳ねる、高域 → まばたき/表情変化、など帯域ごとに割り当てる。
5. `avatar-app.jsx` の `AnalyserNode` 配線（`env`）を流用し、**出力先を「口の開閉」から「モーション/演出」へ差し替える**。

## 作画量メモ

- トークを廃止するなら口の差分（B/C/E/F）はほぼ不要 → **目開け/閉じ（A・D）の2状態 × 25方向 ＝ 50枚** で成立（150枚 → 50枚に削減）。
- 口開け状態を「大きい音のときだけ口を開ける＝曲に合わせて口ずさむ」演出として残すのも、音楽テーマに合う（任意）。

## Claude Code での最初のタスク候補

1. リポジトリを fork / clone し、ローカルで起動して元の挙動を確認する。
2. まず「**音楽反応の核**」だけの最小デモを作る：
   曲を読み込む → ビート検出 → 音符が飛ぶ ＋ キャラ枠がバウンドする
   （キャラ画像はプレースホルダ＝四角や仮画像でOK）。
3. 反応の気持ちよさ・ビート検出の精度を確認してから、自作キャラの25方向シート作成へ進む。

## 補足：開発者の前提（参考）

- 普段は Unity / C# でゲーム開発をしているプログラマー。
- ゲーム開発全般の基礎知識はあるが、shader / material 設定は詳しくない。
- Web フロント（React/JS）は本職ではないので、CSS transform など分かりやすい手法を優先すると進めやすい。
