# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## プロジェクト概要

ブラウザ上のキャラクターアバター。マウスに追従して25方向に振り向き、自動でまばたきする。**Live2D/Spine のような骨格アニメではなく、向き・表情ごとの静止画を `<img>` で重ねて切り替える「画像差し替え式」**である点が全体設計を貫く前提。かつての「ぐるぐる版」「トーク版」は1つのアプリ（`src/avatar-app.jsx`）に統合済み。

進行中の方針: 「口パク」を廃し「**音楽に反応して踊る／演出する**」アバターへ改造中。設計議論は `handover-notes.md` に集約。**踊る版 v1（最小コア）は実装済み**＝読み込んだ曲のビートに**テンポ同期**してキャラが跳ね、**強い拍では大きく跳ね（跳びポ）**、音符パーティクルを飛ばす。さらに**踊り出すと表情が笑顔（B）に変わる**（拍検出中のみ・少し遅れて笑顔・適度に通常顔も差し込む）。詳細は下記「音楽反応（踊る版 v1）」。

## コマンド

```bash
npm install
npm run dev        # Vite dev server (127.0.0.1:5173)、index.html を自動で開く
npm run build      # dist/ を生成（base=/tomari-guruguru/）
npm run preview    # ビルド結果を base=/tomari-guruguru/ で確認（4173）
npm run verify:pages  # dist/ の Pages 成果物を検証（build 後に実行）
```

- dev と build で **base path が異なる**: dev は `/`、build は `/tomari-guruguru/`（GitHub Pages 用）。`preview` は `--base /tomari-guruguru/` を明示しないと壊れる。
- テストランナーは無い。検証は `verify:pages`（dist のHTML/画像の整合）と手動目視。
- マイク入力は `localhost` か HTTPS でのみ動作。Google Fonts は CDN 読み込みのため初回はネット接続が必要。
- 単体テストの仕組みは無いので、変更確認は `npm run dev` でブラウザを開いて目視するのが基本。

## アーキテクチャ

### エントリポイント（単一ページ）
Vite の単一エントリ構成。`vite.config.js` の `rollupOptions.input` は `index.html` のみ:
- `index.html` — 統合アプリ（`src/avatar-app.jsx`）

かつては talk版（`src/talk-app.jsx`）/ ぐるぐる版（`src/app.jsx`）のマルチページだったが、変更点がぼけ不具合の温床になるため **talk版を土台に1本化**した。ぐるぐる版固有の「押下スケール演出」「デバッググリッド」は統合版へ移植済み。音声解析（`AnalyserNode`）の配線は残しつつ、**口パク出力（音量→口シート切替）は無効化**。描画は A=通常（目開け）・D=まばたき（目閉じ）に加え、**踊る/演出版で B=笑顔を流用**（現キャラの B 素材は「笑顔・目閉じ」差し替え済み。本来の B=目開け口中間とは実態が相違＝シート構成は将来整理予定）。口シート C/E/F は未使用。

HTML は `src/tweaks-panel.jsx` を**先に**、続いて `src/avatar-app.jsx` を `<script type="module">` で読む。この読み込み順が重要（下記）。

### 画像フレームの仕組み（最重要）
- 向き = **5列×5行の25方向**。`r{0-4}c{0-4}`（row=上下: 0上→4下、col=左右: 0左→4右）。
- 表情 = **A〜F の6状態**（目開け/閉じ × 口とじ/中間/開け）。
- 画像パス例: `slices2/A/r2c2.webp`（正面・水平・目開け口とじ）。実体は `public/slices2/`。
- **`src/character-config.js` が画像参照の唯一の真実源**。`basePath`・`ext`・グリッド寸法・シート名（A〜F）を集約。キャラ差し替え時はここを書き換えるだけ。`charConfig.src(sheet, r, c)` がパスを生成する。
- レンダリングは「全フレームを `<img>` で重ねて配置し、アクティブな1枚だけ `opacity:1`、他は `0`」方式。統合版 `avatar-app.jsx` は A（通常）/D（まばたき）/B（笑顔）の3シート×25=75枚を描画して切り替える（口パク無効のため口シートC/E/Fは未描画）。
- **表示サイズの仕組み（素材差し替え時の勘所）**: キャラの土台 `<div>` は `width/height: ${charSize * 4/3}vmin`＋`maxWidth/maxHeight: 1200`。各 `<img>` は `width/height: 100%` で土台にフィットするだけなので、**素材のピクセル解像度と画面表示サイズは分離**している（解像度は鮮明さ・メモリにのみ影響）。
  - `vmin`＝ビューポート短辺の1%。画面サイズに追従して拡縮し、`width=height` で正方形を保つ。
  - `* 4/3` は**フレーム内の余白補正**（スライダー値＝体感サイズに対し、枠は一回り大きく取る）。新素材で余白割合が変わればこの係数を調整。
  - マウス追従の中心 `rect.height * 0.45` は**フレーム内の顔の縦位置**基準。顔位置が変われば調整。
  - **素材の native 解像度を変えたら合わせるべきは `maxWidth/maxHeight`**（px絶対の上限＝拡大ボケ防止の天井）。Tweaks の `charSize` の `min/max`（vmin）は「画面占有率＝見た目の大きさ」の意図であり、素材pxとは無関係（見た目を変えたいときだけ触る）。
- **設計判断: スプライトシート（1枚に複数フレームを敷き詰め、CSS `background-position` / Canvas `drawImage` で1セルだけ表示）方式は当面採用しない**。キャラ素材を GPT Image 2 で生成しており、出力ごとに行・列の間隔が揃わず同一行でも数pxずれるため、固定グリッド前提のピボット合わせが破綻する。`tools/slice_character_sheets.py` の component mode はこのズレを吸収する用途なので、「個別フレームへ分割 → `<img>` で重ねる」現行方式を維持する。スプライトシート化を再検討するなら「踊る/演出版」でフレーム要件が固まってから（その際もシートは 4500×4500 のような巨大寸法を避け、GPU上限の実質下限 4096 未満＝2500×2500/500pxセル程度で十分。POTにする必要は無い）。

### 状態の更新ループ
`avatar-app.jsx` の `requestAnimationFrame` ループで:
- マウス座標 → 正規化(-1..1) → スムージング(`smoothing`) → 25グリッドの行/列に量子化 → `setCell`。
- まばたきは独立した `setTimeout` スケジューラ（不規則間隔・二度瞬き・ゆっくり瞬きの確率分岐）。笑顔表示中はまばたきを抑止（笑顔素材が既に目閉じのため）。
- `AnalyserNode` で音量(RMS)と低域エネルギーを解析。音量エンベロープ `env`（attack 速い/release 可変）は音量メーター表示に使用。低域はビート検出に使う（下記「音楽反応」）。`makeAudioEngine()` がファイル音声の解析エンジン（**音源は音声ファイル読み込みのみ**＝マイクUIは外した）。

### 音楽反応（踊る版 v1）
`avatar-app.jsx` のメインループ内に実装。**最小コア**＝テンポ同期バウンド＋強拍アクセント＋音符パーティクル＋踊り中の笑顔表情（ビート駆動の25方向首振りは「標準」段階で先送り、口ずさみも無し）。
- **ビート検出**: 低域(~47-235Hz)エネルギーの**適応しきい値「平均 + k×σ」**の外れ値判定。k は Tweaks「ビート感度」(0..1)で 2.2〜0.5 に写像（高いほど拍が増える）。`AnalyserNode.smoothingTimeConstant=0.2` でキックの瞬発を立てる。比率比較だと連続ベースで二極端になるため σ ベースにしている。**検出した拍は直接モーションを発火せず、下記テンポ位相クロックの補正に回す**。
  - 試行して**不採用**: 低域スペクトラルフラックス（立ち上がり量）検出。密で持続的な低域での自己抑制（密な区間ほど拍が拾えない問題）は解消するが、ベース等の動きで拍以外の単発を多発して「拍以外で跳ぶ／拍で止まる」が悪化したため、絶対レベル方式に戻した。
- **テンポ位相クロック（PLL）**: 内部にメトロノーム（周期 `clockPeriod`・位相 `clockPhase`）を持ち、**バウンドとパーティクルはクロックの拍で発火**。検出拍は周期推定（おおよそ1拍ぶんの間隔だけ採用＝倍/半テンポ取り違え回避）と位相補正（誤差を `[-P/2,P/2]` に折り返し10%だけ寄せる）に使うだけ。これで検出が一時的に抜けても惰性で拍を打ち続け（**拍で止まらない**）、テンポに乗らない単発オンセットは位相補正に吸収される（**拍以外で跳ばない**）。クロックは**再生中かつ直近2秒以内に拍検出があるとき**だけ回す（無音で停止・位相リセット）。
- **モーション（重力積分バウンド）**: 拍ごとに上向き初速 `hopV` を与えるだけの**インパルス＋重力 `hopG` 積分**で `translateY` を毎フレーム連続更新（旧「拍ごとに `hopStart` を今へリセットして放物線を引き直す」方式は、空中で次拍が来ると位置が0へワープしてガクついたため廃止）。滞空 `T` は `clockPeriod` 追従、頂点高 `H` は直近検出拍の `d/σ`（平均から何σ上か）基準。`H`・`T` から `v=4H/T, g=8H/T²` を逆算するので**単発軌道は放物線 `4p(1-p)` と一致**しつつ、空中で次拍が来ても初速差し替えで**位置が連続**（着地ワープ＝ガクつきが消える）。**上昇中(`hopV>0`)は再インパルスを無視**（際限なく上がり続けるのを防ぐ）。`transform` は再描画を避けるため rAF で `charRef.style` に直接書く（押下スケールもここで合成）。
- **強拍アクセント（跳びポ）**: 飛び抜けて強い拍（`d/σ > しきい値`）のとき、クロックとは別タイミングで**大ジャンプを即興で差し込む**。定常リズムは保ったまま要所で抑揚が出る。Tweaks「強拍アクセントの頻度」(0..1)で発火σしきい値(4.5〜2.2)と連発間隔(1800〜900ms)を可変＝**最小でほぼOFF相当**、「強拍アクセントの大きさ」(`accentGain`)で高さ倍率。アクセントは稀なので**上昇中でも今より速い初速なら上書き**してポップさせる。
- **表情（踊り中の笑顔）**: 踊っている間は表情を **B=笑顔**に切り替える（`activeSheet` を `cell` と独立に選択）。起点は**クロックが回っている間だけ**（`clockActive`＝再生中かつ直近2秒以内に拍検出）なので、静かなイントロ/ブレイクでは自動で通常顔に戻る。挙動の勘所:
  - **ウォームアップ**: 踊り出し（`clockActive` 立ち上がり）から `DANCE_WARMUP_MS`（既定2000ms）は通常顔のまま跳ね、経過後に笑顔へ。「踊り始めて少し経つと楽しくなって表情が変わる」演出。笑顔の起点 state `dancing` をウォームアップ後に true にする（変化時のみ setState＝`setCell` と同方針で毎フレーム再描画を避ける）。
  - **笑顔ほどけ**: ずっと笑顔だと単調なので、まばたきと同型の独立スケジューラで**不規則間隔に一瞬だけ通常顔**へ戻す（`smileBreak`）。Tweaks「通常顔に戻る頻度」(`smileBreakFreq` 0..1)で間隔 base を 9000→2000ms に可変（最小でほぼ常時笑顔）。**通常顔は1000〜1800ms キープ**し、笑顔フェーズも最小間隔≥2秒のため、短スパンの笑顔⇔通常パタつきを抑える。
  - 笑顔表示中は**まばたきを抑止**（笑顔素材が既に目閉じ）。Tweaks「踊り中に笑顔」(`smileWhenDancing`)で機能全体を OFF にすると従来の A/D のみ挙動へ戻る。
- **演出（音符パーティクル）**: 拍ごとに音符を飛ばす（`index.html` の `floatUp` キーフレーム）。各粒は `spawnParticles()` で生成。
  - **同時数**: 90%で1個・10%で2個（控えめ）。**duration**: 1.5〜2.5秒（平均約2.0秒）。
  - **色（テンポ連動・パステル）**: 色相は「オレンジ→赤→紫→青→緑」の**長い側270°アーク**を `clockPeriod` に連動（速い拍=オレンジ寄り/遅い拍=緑寄り）。マッピング域は `HUE_PMIN=340〜HUE_PMAX=600`ms（実測 period が速めに寄り上限800まで届かず寒色が出にくいため上限側を狭めた）。各粒に±30°ジッタ。彩度62%・明度は背景で可変（暗72%/明60%）でパステル＆視認性確保。`hsl()` 指定なので背景色変更にも追従。
  - 再生中は idle 浮遊（`bob` クラス）を停止（`<audio>` の play/pause/ended で `playing` 管理）。
- **曲切り替え時の初期化**: 新規ファイル読込（`onFilePick`）で `resetMusicState()` を呼び、前曲の情報を残さない＝適応統計(`lowMean`/`lowVar`)・テンポクロック(`clockPeriod`=500/`clockPhase`)・`lastBeat`/`lastStrength`/`lastAccent`・バウンド(`hopY`/`hopV`/`hopG`)・`env`・踊り/表情(`dancing`/`smileBreak`/`danceStartAt` 等)をリセット（次曲はウォームアップから仕切り直し）。
- **初期音量**: ネイティブ `<audio>` の音量はマウント時に一度だけ `volume=0.5`（初期50%。以後のユーザー操作は維持）。
- **音量と反応の責務分離（設計判断）**: 「音量感度→反応に効く／再生音量→反応に効かない」を実現したいが、再生音量をオーディオグラフで分離するには (a) Analyser 後段の GainNode＋自前スライダー＝**音量UIが2つに重複**、(b) 完全自前プレーヤー＝**ネイティブ再実装のリスク**、のいずれかで不採用。→ **再生音量はネイティブ `<audio controls>` 任せ**にし、代わりに**ダンス側をスケール不変に設計**して音量非依存を担保。ビート検出(平均+k×σ)も跳ね高さ(d/σ)も音量 g 倍で相殺されるため、**再生音量は音量メーター表示にのみ影響し、踊りのタイミング・高さは変わらない**。`micGain`(音量感度)は `level()` に掛かりメーター/`env`に作用。
- **UIレイアウト**: audio プレーヤー＝左下、コントロールバー(ファイル選択＋音量メーター)＝下中央、Tweaks＝右下（基盤 `tweaks-panel.jsx` は右下固定のまま）。互いに重ならないよう分散。
- **モバイル対応（狭幅・iOS Safari）**: 縦長スマホで UI がキャラを覆う/操作不能になる問題への対処。いずれも**基盤 `tweaks-panel.jsx` は無改変**で、`index.html` のグローバル `<style>`（`@media (max-width:480px)` と `.twk-panel` 上書き）＋ `avatar-app.jsx` 側の調整で実現。
  - **下中央バーの縦潰れ回避**: 旧 `left:50%`＋`translateX(-50%)`（幅指定なし）は shrink-to-fit 幅が「左50%〜右端＝50vw」で頭打ちになり、狭幅でファイル名ラベルが1文字ずつ縦に潰れていた。**透明な全幅センタリング容器(`left:0;right:0;justifyContent:center;pointerEvents:none`)＋中のピル(`maxWidth:100%;pointerEvents:auto`)** 構成に変更し、ラベルは `whiteSpace:nowrap;flexShrink:0`。
  - **音量メーターを狭幅で非表示**: メーター群に `vol-meter` クラス。`@media(max-width:480px){.vol-meter{display:none!important}}`（インラインの `display:flex` に勝たせるため `!important` 必須）。再生音量は audio プレーヤー側にもあるため省いても支障なし。
  - **audio とバーの衝突回避**: 狭幅では audio がほぼ全幅になり下中央バーと重なる。バー容器に `ctrl-bar`＋（ファイル読込中のみ）`has-audio` クラスを付与し、`@media(max-width:480px){.ctrl-bar.has-audio{bottom:88px!important}}` で**読込後だけ**ピルを audio の上へ持ち上げる（広い画面では発火せず元レイアウトのまま）。audio 自体も `maxWidth:calc(100vw-40px)` で溢れ防止。
  - **Tweaks の ✕ が画面外に隠れる問題**: 基盤の `.twk-panel` は `max-height:calc(100vh - 32px)`＋`bottom:16px`。iOS Safari の `100vh` はツールバー裏を含むため可視領域より高くなり、下端基準で上に伸びてヘッダー(✕)が URL バー裏に隠れた。→ `index.html` で `.twk-panel{max-height:calc(100dvh - 32px)!important}` と**可視領域基準の `dvh`** に上書き（基盤は触らない。clamp 用の `window.innerHeight` は元から可視高なので整合）。
  - **Tweaks をパネル外タップで閉じる**: ✕ に気付きにくい初見ユーザー向け。`avatar-app.jsx` の `document` capture `pointerdown` リスナで、`.twk-panel` が存在(=開)かつタップ先がパネル外なら `window.postMessage({type:'__deactivate_edit_mode'})` を送る（パネルはこのメッセージで閉じる作り＝基盤無改変で実現）。**バックドロップを置かない**方式なので開いたままでもキャラのマウス追従プレビューは生きる（モーダル化しない）。

### Tweaks パネルと EDITMODE ブロック（注意が必要な仕組み）
`src/tweaks-panel.jsx` は外部プロトタイピング基盤（"omelette" scaffold）由来。**import されず、`Object.assign(window, {...})` で `useTweaks`/`TweaksPanel`/`TweakSlider` 等をグローバル公開する**。だから各アプリ JSX はこれらを import 無しで使える。HTML での読み込み順（tweaks-panel が先）が崩れると未定義になる。

各アプリ冒頭の `/*EDITMODE-BEGIN*/{...}/*EDITMODE-END*/` は tweak のデフォルト値。`useTweaks` の `setTweak` は `window.parent.postMessage({type:'__edit_mode_set_keys'})` でホストへ送り、ホストがこの JSON ブロックをディスク上に書き戻す前提。スタンドアロンで `npm run dev` する分にはメモリ内 state として動くだけで永続化はされない。この protocol/CSS は基盤由来なので通常いじらない。

## キャラ画像の生成（差し替え）
`tools/slice_character_sheets.py` が 4500×4500 の5×5シート6枚を 1200×1200 の個別フレームへスライスする。**ffmpeg/ffprobe が必要**。

- 単純な900pxセル切りではなく **component mode**（シート全体の連結成分を検出して25セルへ割当）が既定。隣キャラ混入・髪見切れを防ぐため。
- グレー背景の半透明残りは `--remove-gray-residue` で除去。
- 詳細手順・検証チェックリスト・ログの見方（`large=25`、各セル `1:xxxxx`）は `docs/新キャラ差し替え手順.md` を参照。
